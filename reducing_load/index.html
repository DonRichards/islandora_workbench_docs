Reducing Workbench's impact on Drupal

Workbench can put a substantial stress on Drupal. In some cases, this stress can lead to instability and errors. Workbench offers two main ways to reduce this stress.

The first is by telling Workbench to pause between each REST request it makes to Drupal. There are two types of pause, 1) basic pause and 2 "adaptive" pause.

Basic pause

Using the pause configuration setting will tell Workbench to pause execution before every 'POST', 'PUT', 'PATCH', and 'DELETE' request. 

pause: 2

Adaptive pause

Adaptive pause only pauses execution between requests if Workbench detects that Drupal is slowing down. It does does this by monitoring Drupal's average response time (to be specific, the average of the last 20 HTTP responses). If descrease in Drupal's response time reaches a specific threshold, Workbench's adaptive pause will kick in and temporarily halt execution to allow Drupal to catch up.

The threshold that needs to be met is configured using the adaptive_pause_threshold option. This setting's default value is 2, which means that the adaptive pause will kick in if the response time for the last request Workbench makes to Drupal is 2 times (double) the average of the last 20 requests. The number of seconds that Workbench will pause is determined by the adaptive_pause setting. Since adaptive_pause_threshold has a default value (2) but adaptive_pause does not, you would only ever use adaptive_pause_threshold if you want to override the default value. Therefore, both of these configurations are valid:

adaptive_pause: 3

and

adaptive_pause: 2
adaptive_pause_threshold: 4

It might take trial and error to determine the optimal combination of these two settings for your Drupal server. A third configuration setting, log_response_time: true, will log the response time for every request Workbench makes to Drupal. Including this setting in your configuration will add a lot of entries to your Workbench log, but the data might help you get a sense for the effects that log_response_time adaptive_pause and adaptive_pause_threshold are having on your Drupal server.

Caching

The second way that Workbench reduces stress on Drupal is by caching. By default, Workbench caches results to some types of requests is makes to Drupal, such as requests for field configurations or for checks for the existence of taxonomy terms. This speeds up Workbench considerably. You should never have to disable this caching, but if you do (for example, if you are asked to during troubleshooting), you can do so by including the following setting in your configuration file:

enable_http_cache: false
