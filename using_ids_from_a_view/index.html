You can use a new or existing View to generate a list of node IDs that Workbench can use in update, delete, add_media, export_csv, and delete_media_by_node tasks instead of an input file. This is done using a get_data_from_view task. A sample configuration file looks like this:

task: get_data_from_view
host: "http://localhost:8000/"
view_path: '/427test'
username: admin
password: islandora
data_from_view_file_path: /tmp/427.csv

The view_path setting should contain the value of the "Path" option in the Views configuration page's "Path settings" section. The data_from_view_file_path is the location where you want your CSV file saved.

In the View configuration page:

  1. Add a "REST export" display.
  2. Under "Format" > "Serializer" > "Settings", choose "json".
  3. In the View "Fields" settings, leave "The selected style or row format does not use fields" as is (see explanation below).
  4. Under "Path", add a path where your REST export will be accessible to Workbench. The URL for REST export you will use in the input_csv configuration setting will be the value of this path appended to your Drupal's hostname. For example, if the "Path" value you have chose is my-rest-export, and your Drupal's hostname is http://example.org, the value of your view_path setting should be /my-rest-export.
  5. Under "Pager" > "Items to display", choose "Paged output, mini pager". In "Pager options" choose 10 items to display.
  6. Under "Path settings" > "Access", choose "Permission" and "View published content". Under "Authentication", choose "basic_auth".

Warning

If your View includes nodes that you do not want to be seen by anonymous users, of if it contains unpublished nodes, adjust the access permissions settings appropriately.

REST export Views displays don't use fields in the same way that other Views displays do. In fact, Drupal says within the Views user interface that for REST export displays, "The selected style or row format does not use fields." Instead, these displays export the entire node in JSON format. Workbench iterates through all fields on the node JSON that start with field_ and includes those fields, plus node_id and title, in the output CSV.

To test your REST export, in your browser, go to the URL. You should see raw JSON (or formatted JSON if your browser renders JSON to be human readable) that lists the nodes in your View.
